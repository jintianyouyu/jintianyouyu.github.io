feroxbuster-目录扫描工具

器字篇
信息收集-目录扫描
发布日期:   2022-08-18
文章字数:   2.2k
阅读次数:  
  • 工具下载地址：https://github.com/epi052/feroxbuster

工具介绍

  • 目录扫描工具，目录字典需要用自己的，使用 -w 参数手动指定。
  • 至于说的速度快、很好用之类的，还没实战测试

参数介绍

x86_64-windows-feroxbuster>feroxbuster.exe -h
feroxbuster 2.7.1
Ben 'epi' Risher (@epi052)
A fast, simple, recursive content discovery tool.

USAGE:
    feroxbuster [OPTIONS]

OPTIONS:
    -h, --help       Print help information
    -V, --version    Print version information

Target selection:	//扫描目标选择：三种方式 
        --resume-from <STATE_FILE>    State file from which to resume a partially complete scan (ex. --resume-from ferox-1606586780.state)	//扫描到一般保存的扫描日志文件（查看Output settings参数部分你会明白）
        --stdin                       Read url(s) from STDIN	//stdin标准输入（键盘输入到缓冲区中的内容）--不知道怎么用
    -u, --url <URL>                   The target URL (required, unless [--stdin || --resume-from] used)	//常规用法，如果不使用前两种方式，此参数必须有

Composite settings:	//没懂，不会用
        --burp           Set --proxy to http://127.0.0.1:8080 and set --insecure to true
        --burp-replay    Set --replay-proxy to http://127.0.0.1:8080 and set --insecure to true
        --smart          Set --extract-links, --auto-tune, --collect-words, and --collect-backups to true
        --thorough       Use the same settings as --smart and set --collect-extensions to true

Proxy settings:	//配置代理
    -p, --proxy <PROXY>                    Proxy to use for requests (ex: http(s)://host:port, socks5(h)://host:port)	//设置代理
    -P, --replay-proxy <REPLAY_PROXY>      Send only unfiltered requests through a Replay Proxy, instead of all requests	//只通过Replay Proxy发送未过滤的请求，而不是所有的请求
    -R, --replay-codes <REPLAY_CODE>...    Status Codes to send through a Replay Proxy when found (default: --status-codes value)	//通过重放代理发送的状态码(默认值:--status-codes)

Request settings:	//请求设置
    -a, --user-agent <USER_AGENT>           Sets the User-Agent (default: feroxbuster/2.7.1)	//设置用户代理(默认:feroxbuster/2.7.1)
    -A, --random-agent                      Use a random User-Agent	//使用随机的user-agent
    -b, --cookies <COOKIE>...               Specify HTTP cookies to be used in each request (ex: -b stuff=things)	//添加网站的cookie进行扫描
        --data <DATA>                       Request's Body; can read data from a file if input starts with an @ (ex: @post.bin)
    -f, --add-slash                         Append / to each request's URL	//对请求设置后使用此参数添加到每个请求中（自己理解）
    -H, --headers <HEADER>...               Specify HTTP headers to be used in each request (ex: -H Header:val -H 'stuff: things')	//可自定义在每个请求头中添加HTTP头
    -m, --methods <HTTP_METHODS>...         Which HTTP request method(s) should be sent (default: GET)	//设置发送的HTTP请求方法：GET（默认），可自行指定
    -Q, --query <QUERY>...                  Request's URL query parameters (ex: -Q token=stuff -Q secret=key)	//查询
    -x, --extensions <FILE_EXTENSION>...    File extension(s) to search for (ex: -x php -x pdf js)	//添加需要搜索的文件扩展名

Request filters:	//过滤扫描
        --dont-scan <URL>...    URL(s) or Regex Pattern(s) to exclude from recursion/scans	//排除不扫描/递归扫描的URL

Response filters:	//响应过滤器
    -C, --filter-status <STATUS_CODE>...          Filter out status codes (deny list) (ex: -C 200 -C 401)	//过滤掉对应的响应代码
        --filter-similar-to <UNWANTED_PAGE>...    Filter out pages that are similar to the given page (ex. --filter-similar-to http://site.xyz/soft404)	//过滤掉与给定页面类似的页面
    -N, --filter-lines <LINES>...                 Filter out messages of a particular line count (ex: -N 20 -N 31,30)	//过滤掉特定行数的消息（未测试）
    -s, --status-codes <STATUS_CODE>...           Status Codes to include (allow list) (default: 200 204 301 302 307 308 401 403 405)	//允许显示状态码过滤
    -S, --filter-size <SIZE>...                   Filter out messages of a particular size (ex: -S 5120 -S 4927,1970)	//过滤掉特定大小的消息
    -W, --filter-words <WORDS>...                 Filter out messages of a particular word count (ex: -W 312 -W 91,82)	//过滤掉特定字数的消息(未测试)
    -X, --filter-regex <REGEX>...                 Filter out messages via regular expression matching on the response's body (ex: -X '^ignore me$')	//通过在响应体上匹配正则表达式过滤掉消息(未测试)

Client settings:
    -k, --insecure             Disables TLS certificate validation in the client	//在客户端禁用TLS证书验证
    -r, --redirects            Allow client to follow redirects	//允许客户端遵循重定向
    -T, --timeout <SECONDS>    Number of seconds before a client's request times out (default: 7)	//客户端请求超时前的秒数(默认值:7)

Scan settings:	//扫描设置
        --auto-bail                    Automatically stop scanning when an excessive amount of errors are encountered	//遇到过多错误时停止扫描
        --auto-tune                    Automatically lower scan rate when an excessive amount of errors are encountered	//当遇到过多的错误时，自动降低扫描速率
    -d, --depth <RECURSION_DEPTH>      Maximum recursion depth, a depth of 0 is infinite recursion (default: 4)	//最大递归深度，默认为4，如果设置为0则无限递归
    -D, --dont-filter                  Don't auto-filter wildcard responses
    -e, --extract-links                Extract links from response body (html, javascript, etc...); make new requests based on findings
        --force-recursion              Force recursion attempts on all 'found' endpoints (still respects recursion depth)
    -L, --scan-limit <SCAN_LIMIT>      Limit total number of concurrent scans (default: 0, i.e. no limit)	//限制并发扫描的总数(默认为0，即不限制)
    -n, --no-recursion                 Do not scan recursively	//不进行递归扫描
        --parallel <PARALLEL_SCANS>    Run parallel feroxbuster instances (one child process per url passed via stdin)	//（高端用法，用到再说）
        --rate-limit <RATE_LIMIT>      Limit number of requests per second (per directory) (default: 0, i.e. no limit)	//限制每秒请求数(每个目录)(默认:0，即不限制)
    -t, --threads <THREADS>            Number of concurrent threads (default: 50)	//线程数设置，默认为50
        --time-limit <TIME_SPEC>       Limit total run time of all scans (ex: --time-limit 10m)	//限制所有扫描的总运行时间，如：可设置只扫10分钟
    -w, --wordlist <FILE>              Path to the wordlist	//扫描字典路径

Dynamic collection settings:
    -B, --collect-backups                     Automatically request likely backup extensions for "found" urls
    -E, --collect-extensions                  Automatically discover extensions and add them to --extensions (unless they're in --dont-collect)
    -g, --collect-words                       Automatically discover important words from within responses and add them to the wordlist
    -I, --dont-collect <FILE_EXTENSION>...    File extension(s) to Ignore while collecting extensions (only used with --collect-extensions)

Output settings:	//输出设置
        --debug-log <FILE>    Output file to write log entries (use w/ --json for JSON entries)
        --json                Emit JSON logs to --output and --debug-log instead of normal text
        --no-state            Disable state output file (*.state)	//禁用状态输出文件(*.state)
    -o, --output <FILE>       Output file to write results to (use w/ --json for JSON entries)	//将结果写入文件
    -q, --quiet               Hide progress bars and banner (good for tmux windows w/ notifications)	//隐藏扫描进度条（没有这个看着挺难受）
        --silent              Only print URLs + turn off logging (good for piping a list of urls to other commands)	//只打印url +关闭日志记录(还是看着难受)
    -v, --verbosity           Increase verbosity level (use -vv or more for greater effect. [CAUTION] 4 -v's is probably too much)

工具使用

  • 工具的运行，使用 -u 参数，默认使用以下标注的功能，与 dirmap 工具最开始的显示相同，添加功能参数，此处会列出来对应使用的参数功能

Request settings：请求设置

  • -A：使用随机的user-agent

    • 程序自动随机修改user-agent，进行反爬绕过
    • 以上图相比 user-agent 由默认变成了随机（Random)

  • -f：对“请求设置”后，如修改的user-agent等参数，可使用此参数添加到每个请求中

  • -m：设置发送的HTTP请求方法：GET（默认），可自行指定

  • -x：指定要搜索的文件扩展名

Request filters：过滤扫描

  • –dont-scan：排除不扫描/递归扫描的URL

Response filters：响应过滤器

  • -C：过滤掉对应的响应代码，对于不必要显示的响应代码都进行过滤，此参数不能与 -s 参数一起使用
    • 简单说：手动设置后将是设置一个过滤一个

  • –filter-similar-to …：过滤掉与给定页面类似的页面
  • 例如：–filter-similar-to http://site.xyz/soft404

  • -s：允许显示状态码过滤，此参数不能与 -C 参数一起使用
  • 默认显示的状态码：200 204 301 302 307 308 401 403 405

  • -S：过滤掉特定大小的消息
    • 例如：-S 5120 -S 4927,1970，此处的过滤大小应该是 Byte

Scan settings：扫描设置

  • -d：最大递归深度，默认为4，如果设置为0则无限递归

  • 此参数可设置递归层数

  • -n：不进行递归扫描

  • -t：线程数设置，默认为50

工具使用总结

  • 总体而言此工具功能比较多也比较强大，其中还有很多的高级功能，可自行开发，对于此工具列出个常用的命令，使用时可直接赋值命令进行修改即可

    ./feroxbuster -u http://www.gshfns.com/ -w dict_mode_dict.txt -A -b stuff=things -m GET/POST  -s 200 -S 1143 -d 2 -t 100
* -w ：给定自己准备好的字典，程序本身不带字典
* -A ：使用随机user-agent，防止反爬拦截
* -b ：添加cookie参数，可用于登录扫描
* -m ：自定义请求方式GET、POST或者其他
* -s ：允许显示的请求状态码，此参数与-C对立使用
* -S ：过滤特定大小的数据包，大小单位为byte
* -d ：设置扫描递归深度，-n：表示不使用递归扫描（一般都用-n)
* -t ：线程数默认50，调节扫描速度

  • 工具使用可直接复制以上参数进行修改，若要设置更多功能参数看以上详解

    • 此处参数按照个人习惯摘取，仁者见仁智者见智
